How It Works

A deep dive into the engine's internals — from board representation to search.

🧩

Board Representation

64‑int array with piece codes, side to move, castling rights, en passant, and move clocks.

int squares[64];   int side_to_move;
⚡

Move Generation

Pseudo‑legal move generation with a legality filter using copy‑make and king‑safety checks.

add_pawn_moves() · add_knight_moves()
🔍

Negamax Search

Alpha‑beta pruning with MVV‑LVA move ordering. Depth‑4 search, configurable.

negamax(depth, α, β) · order_moves()
📊

PeSTO Evaluation

Material + piece‑square tables for middlegame and endgame, interpolated by phase.

mg_score + eg_score · phase = 24

// Negamax with alpha-beta pruning
static int negamax(Board& b, int depth, int alpha, int beta) {
  nodes_count++;
  if (depth == 0) return evaluate(b);
  auto moves = generate_legal_moves(b);
  if (moves.empty()) {
    return is_in_check(b, b.side_to_move) ? -100000 + (100 - depth) : 0;
  }
  order_moves(moves, b);
  for (const auto& m : moves) {
    Board copy = b;
    make_move(copy, m);
    int score = -negamax(copy, depth - 1, -beta, -alpha);
    if (score >= beta) return beta;
    if (score > alpha) alpha = score;
  }
  return alpha;
}
